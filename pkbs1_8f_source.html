4 C> THIS SUBROUTINE STORES A SPECIFIED INTEGER VALUE INTO A
5 C> SPECIFIED LOCATION WITHIN SECTION 1 OF THE BUFR MESSAGE STORED IN
6 C> ARRAY MBAY, OVERWRITING THE VALUE PREVIOUSLY STORED AT THAT
7 C> LOCATION. IT WILL WORK ON ANY MESSAGE ENCODED USING BUFR EDITION
8 C> 2, 3 OR 4. THE START OF THE BUFR MESSAGE (I.E. THE STRING "BUFR")
9 C> MUST BE ALIGNED ON THE FIRST FOUR BYTES OF MBAY, AND THE LOCATION
10 C> WITHIN WHICH TO STORE THE VALUE IS SPECIFIED VIA THE MNEMONIC
11 C> S1MNEM, AS EXPLAINED IN FURTHER DETAIL BELOW.
12 C>
13 C> PROGRAM HISTORY LOG:
14 C> 2005-11-29 J. ATOR -- ORIGINAL AUTHOR
15 C> 2006-04-14 D. KEYSER -- ADDED OPTIONS FOR 'MTYP', 'MSBT', 'YEAR',
16 C> 'MNTH', 'DAYS', 'HOUR', 'YCEN' AND 'CENT'
17 C>
18 C> USAGE: PKBS1 (IVAL, MBAY, S1MNEM)
19 C> INPUT ARGUMENT LIST:
20 C> IVAL - INTEGER: VALUE TO BE STORED
21 C> MBAY - INTEGER: *-WORD PACKED BINARY ARRAY CONTAINING
22 C> BUFR MESSAGE PRIOR TO STORING IVAL
23 C> S1MNEM - CHARACTER*(*): MNEMONIC SPECIFYING LOCATION WHERE IVAL
24 C> IS TO BE STORED WITHIN SECTION 1 OF BUFR MESSAGE:
25 C> 'BMT' = BUFR MASTER TABLE
26 C> 'OGCE' = ORIGINATING CENTER
27 C> 'GSES' = ORIGINATING SUBCENTER
28 C> (NOTE: THIS VALUE IS STORED ONLY IN

30 C> 'USN' = UPDATE SEQUENCE NUMBER
31 C> 'MTYP' = DATA CATEGORY
32 C> 'MSBTI' = DATA SUBCATEGORY (INTERNATIONAL)
33 C> (NOTE: THIS VALUE IS STORED ONLY IN
34 C> BUFR EDITION 4 MESSAGES!)
35 C> 'MSBT' = DATA SUBCATEGORY (LOCAL)
36 C> 'MTV' = VERSION NUMBER OF MASTER TABLE
37 C> 'MTVL' = VERSION NUMBER OF LOCAL TABLES
38 C> 'YCEN' = YEAR OF CENTURY (1-100)
39 C> (NOTE: THIS VALUE IS STORED ONLY IN

55 C>
