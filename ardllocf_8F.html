subroutine ardllocf

THIS SUBROUTINE FREES ANY MEMORY THAT WAS DYNAMICALLY ALLOCATED BY PREVIOUS CALLS TO BUFR ARCHIVE LIBRARY ROUTINES ARALLOCF OR ARALLOCC.

NOTE THAT THIS SUBROUTINE IS CALLED WITHIN BUFR ARCHIVE LIBRARY SUBROUTINE EXITBUFR AS PART OF THE PROCESS TO RESET THE LIBRARY AND PREPARE IT FOR POTENTIAL RE-ALLOCATION OF NEW ARRAY SPACE VIA ONE OR MORE SUBSEQUENT CALLS TO SUBROUTINES ISETPRM AND OPENBF. THIS SUBROUTINE SHOULD ONLY BE CALLED DIRECTLY BY AN APPLICATION PROGRAM IF THE PROGRAM IS COMPLETELY FINISHED WITH ALL CALLS TO ALL OTHER BUFR ARCHIVE LIBRARY ROUTINES, BECAUSE THE MEMORY FREED HEREIN WILL RENDER THE LIBRARY AS EFFECTIVELY UNUSABLE FOR THE REMAINDER OF THE LIFE OF THE APPLICATION PROGRAM. HOWEVER, THIS MAY BE A USEFUL OPTION FOR APPLICATION PROGRAMS WHICH WANT TO MOVE ON TO OTHER UNRELATED TASKS WITHOUT CONTINUING TO TIE UP A SIGNIFICANT AMOUNT OF DYNAMICALLY-ALLOCATED HEAP MEMORY RELATED TO THIS LIBRARY. OTHERWISE, ALL SUCH MEMORY WILL BE FREED AUTOMATICALLY ONCE THE APPLICATION PROGRAM TERMINATES.

USAGE: CALL ARDLLOCF

REMARKS: THIS ROUTINE CALLS: ARDLLOCC THIS ROUTINE IS CALLED BY: EXITBUFR Also called by application programs.
