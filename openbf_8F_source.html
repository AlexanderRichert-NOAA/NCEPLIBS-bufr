4 C> THIS SUBROUTINE NORMALLY (I.E. EXCEPT WHEN INPUT ARGUMENT
5 C> IO IS 'QUIET') IDENTIFIES A NEW LOGICAL UNIT TO THE BUFR ARCHIVE
6 C> LIBRARY SOFTWARE FOR INPUT OR OUTPUT OPERATIONS. HOWEVER, THE
7 C> FIRST TIME IT IS CALLED, IT ALSO FIGURES OUT SOME IMPORTANT
8 C> INFORMATION ABOUT THE LOCAL MACHINE ON WHICH THE SOFTWARE IS BEING
9 C> RUN (VIA A CALL TO BUFR ARCHIVE LIBRARY SUBROUTINE WRDLEN), AND IT
10 C> ALSO INITIALIZES ARRAYS IN MANY BUFR ARCHIVE LIBRARY COMMON BLOCKS
11 C> (VIA A CALL TO BUFR ARCHIVE LIBRARY SUBROUTINE BFRINI). UP TO 32
12 C> LOGICAL UNITS CAN BE CONNECTED TO THE BUFR ARCHIVE LIBRARY SOFTWARE
13 C> AT ANY ONE TIME.
14 C>
15 C> NOTE: IF IO IS PASSED IN AS 'QUIET', THEN OPENBF PERFORMS ONLY ONE
16 C> FUNCTION - IT SIMPLY SETS THE "DEGREE OF PRINTOUT" SWITCH IPRT (IN
17 C> COMMON BLOCK /QUIET/) TO THE VALUE OF INPUT ARGUMENT LUNDX,
18 C> OVERRIDING ITS PREVIOUS VALUE. A DEFAULT IPRT VALUE OF 0 (I.E.
19 C> "LIMITED PRINTOUT") IS SET DURING THE FIRST CALL TO THIS ROUTINE,
20 C> BUT THIS OR ANY OTHER IPRT VALUE MAY BE SET AND RESET AS OFTEN AS
21 C> DESIRED VIA SUCCESSIVE CALLS TO OPENBF WITH IO = 'QUIET'.
22 C> IN ALL SUCH CASES, OPENBF SIMPLY (RE)SETS IPRT AND THEN RETURNS
23 C> WITHOUT ACTUALLY OPENING ANY FILES. THE DEGREE OF PRINTOUT
24 C> INCREASES AS IPRT INCREASES FROM "-1" TO "0" TO "1" TO "2".
25 C>

65 C> USAGE: CALL OPENBF (LUNIT, IO, LUNDX)
66 C> INPUT ARGUMENT LIST:
67 C> LUNIT - INTEGER: FORTRAN LOGICAL UNIT NUMBER FOR BUFR FILE
68 C> (UNLESS IO IS 'QUIET', THEN A DUMMY)
69 C> IO - CHARACTER*(*): FLAG INDICATING HOW LUNIT IS TO BE
70 C> USED BY THE SOFTWARE:
71 C> 'IN' = input operations with table processing
72 C> 'INX' = input operations w/o table processing
73 C> 'OUX' = output operations w/o table processing
74 C> 'OUT' = output operations with table processing
75 C> 'SEC3' = same as 'IN', except use Section 3 of input
76 C> messages for decoding rather than dictionary
77 C> table information from LUNDX; in this case
78 C> LUNDX is ignored, and user must provide
79 C> appropriate BUFR master tables within
80 C> directory specified by a subsequent call
81 C> to subroutine MTINFO
82 C> 'NODX' = same as 'OUT', except don't write dictionary
83 C> (i.e. DX) table messages to LUNIT
84 C> 'APN' = same as 'NODX', except begin writing at end
85 C> of file ("append")
86 C> 'APX' = same as 'APN', except backspace before
87 C> appending
88 C> 'NUL' = same as 'OUT', except don't write any
89 C> messages whatsoever to LUNIT (e.g. when
90 C> subroutine WRITSA is to be used)
91 C> 'INUL' = same as 'IN', except don't read any
92 C> messages whatsoever from LUNIT (e.g. when
93 C> subroutine READERME is to be used)
94 C> 'QUIET' = LUNIT is ignored, this is an indicator
95 C> that the value for IPRT in COMMON block
96 C> /QUIET/ is being reset (see LUNDX)
97 C> 'FIRST' = calls bfrini and wrdlen as a prelude to user
98 c resetting of bufrlib parameters such as
99 c missing value or output block type
100 C> LUNDX - INTEGER: IF IO IS NOT 'QUIET':
101 C> FORTRAN logical unit number containing
102 C> dictionary table information to be used in
103 C> reading/writing from/to LUNIT (depending
104 C> on the case); may be set equal to LUNIT if
105 C> dictionary table information is already
106 C> embedded in LUNIT
107 C> IF IO IS 'QUIET':
108 C> Indicator for degree of printout:
109 C> -1 = NO printout except for ABORT
110 C> messages
111 C> 0 = LIMITED printout (default)
112 C> 1 = ALL warning messages are printed
113 C> out
114 C> 2 = ALL warning AND informational
115 C> messages are printed out
116 C> (Note: this does not change until OPENBF
117 C> is again called with IO equal to
118 C> 'QUIET')
119 C>
120 C> INPUT FILES:
