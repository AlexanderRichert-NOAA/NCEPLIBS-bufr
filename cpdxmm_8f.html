subroutine cpdxmm (   LUNIT)

BEGINNING AT THE CURRENT FILE POINTER LOCATION WITHIN LUNIT, THIS SUBROUTINE READS A COMPLETE DICTIONARY TABLE (I.E.

ONE OR MORE ADJACENT BUFR DX (DICTIONARY) MESSAGES) INTO MODULE MSGMEM.

PROGRAM HISTORY LOG: 2009-03-23 J. ATOR – ORIGINAL AUTHOR 2012-09-15 J. WOOLLEN – MODIFIED FOR C/I/O/BUFR INTERFACE; REPLACED FORTRAN BACKSPACE WITH C BACKBUFR 2014-12-10 J. ATOR – USE MODULES INSTEAD OF COMMON BLOCKS

USAGE: CALL CPDXMM (LUNIT) INPUT ARGUMENT LIST: LUNIT - INTEGER: FORTRAN LOGICAL UNIT NUMBER FOR BUFR FILE

REMARKS:

THE FOLLOWING VALUES ARE STORED WITHIN MODULE MSGMEM BY THIS SUBROUTINE:

LDXM = number of array words filled within MDX

MDX(I=1,LDXM) = DX dictionary messages for use in decoding data messages stored within MSGS array (in MODULE MSGMEM)

NDXM = number of DX dictionary messages within MDX

IPDXM(I=1,NDXM) = pointer to first word of (I)th message within MDX

NDXTS = number of DX dictionary tables represented by messages within MDX

IFDXTS(J=1,NDXTS) = sequential number of first message within MDX which is part of (J)th dictionary table

ICDXTS(J=1,NDXTS) = count of consecutive messages within MDX (beginning with IFDXTS(J)) which constitute (J)th dictionary table

IPMSGS(J=1,NDXTS) = sequential number of first data message within MSGS array (in MODULE MSGMEM) to which (J)th dictionary table applies

LDXTS = current dictionary table that is in scope (i.e. a number between 1 and NDXTS)

THIS ROUTINE CALLS: BORT ERRWRT IDXMSG IUPBS3 NMWRD RDMSGW THIS ROUTINE IS CALLED BY: UFBMEM Not normally called by application programs.
