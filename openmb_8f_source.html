4 C> THIS SUBROUTINE OPENS AND INITIALIZES A NEW BUFR MESSAGE
5 C> WITHIN MEMORY. IT SHOULD ONLY BE CALLED WHEN LOGICAL UNIT LUNIT
6 C> HAS BEEN OPENED FOR OUTPUT OPERATIONS. IT IS SIMILAR TO BUFR
7 C> ARCHIVE LIBRARY SUBROUTINE OPENMG, HOWEVER UNLIKE OPENMG, IT WILL
8 C> NOT OPEN A NEW MESSAGE IF THERE IS ALREADY A BUFR MESSAGE OPEN
9 C> WITHIN MEMORY FOR THIS LUNIT WHICH HAS THE SAME SUBSET AND JDATE
10 C> VALUES (IN WHICH CASE IT DOES NOTHING AND RETURNS TO THE CALLING
11 C> ROUTINE/PROGRAM). OTHERWISE, IF THERE IS ALREADY A BUFR MESSAGE
12 C> OPEN WITHIN MEMORY FOR THIS LUNIT BUT WHICH HAS A DIFFERENT SUBSET
13 C> OR JDATE VALUE, THEN THAT MESSAGE WILL BE CLOSED AND FLUSHED TO
14 C> LUNIT BEFORE OPENING THE NEW ONE.
15 C>
