23 ### debufr
24 
25 This program decodes a BUFR file and writes a verbose listing of the contents to the file specified
26 via the -o option.
27 If a [DX BUFR Tables](@ref dfbftab) file is specified (using the -f option) or if the specified BUFR file
28 contains an embedded DX BUFR tables message as the first message in the file, then this information is
29 used to decode the data messages in the file. Otherwise, or whenever the -m option is specified,
30 [master BUFR tables](@ref dfbfmstab) are read and used to decode the data messages in the file.
31 
32 <pre>
33 Usage:
34 
35  debufr [-v] [-h] [-b] [-c] [-m] [-o outfile] [-t tabledir] [-f tablefil] [-p prmstg] bufrfile
36 
37  where:
38  -v prints version information and exits
39 
40  -h prints program help and usage information and exits
41 
42  -b specifies the "basic" option, meaning that only the
43  information in Sections 0-3 will be decoded from each
44  BUFR message in the bufrfile, and no attempt will be
45  made to decode the data in Section 4
46 
47  -c specifies that code and flag table meanings should not
48  be read from master BUFR tables and included in the
49  output; otherwise this feature is enabled by default
50 
51  -m specifies that master BUFR tables will be used to
52  decode the data messages in the file, regardless of
53  whether it contains any embedded DX BUFR table
54  messages. This option can be used to view the actual
55  contents of DX BUFR table messages, which otherwise
56  would not be printed in the output listing.
57 
58  outfile [path/]name of file to contain verbose output listing.
59  The default is "bufrfilename.debufr.out" in the current
60  working directory, where bufrfilename is the basename of
61  the bufrfile (i.e. bufrfile with any preceding [path/]
62  removed).
63 
64  tabledir [path/]name of directory containing tables to be used
65  for decoding. This directory contains the DX BUFR
66  tables file to be used (if one was specified
67  via the -f option), or it may contain all of the master
68  BUFR tables when these are being used to decode a
69  file. If unspecified, the default directory location is
70  the defined value of the MASTER_TABLE_DIR macro when the
71  utility was built.
72 
73  tablefil file within tabledir containing DX BUFR tables
74  to be used for decoding.
75 
76  prmstg string of comma-separated PARAMETER=VALUE pairs, up to a
77  maximum of 20. For each pair, the dynamic allocation
78  PARAMETER will be set to VALUE within the underlying
79  BUFRLIB software, overriding the default value that
80  would otherwise be used. A complete list of parameters
81  that can be dynamically sized is included within the
82  documentation for BUFRLIB function isetprm().
83 
84  bufrfile [path/]name of BUFR file to be decoded
85 </pre>
86 
87 See the source code at debufr.c and debufr.f

95 ### readbp
96 
97 A utility to read prepbufr files which prints each report one at a time, or jumps to a report with characteristics defined by various keys such as report type, subset type, xy locagtion, station id, etc. Keys can be entered as arguments to the program or entered while the program is running. Basic operation is to print one report at a time with the default being starting at the beginning and continuing until the end or the user enters 'q'. The following summary is printed if the program is run without arguments.
98 
99 <pre>
100  Usage: readbp <-s> <-w> <m> <-k> <-r> <-d> <-n> <-h> prep bufrfile
101 
102  Search filter and/or print prepbufr reports in various ways
103 
104  -s "station_id " print reports where "station_id" matches the report id up to the len of "station_id"
105  -w "x1 x2 y1 y2" print reports within a lon/lat box
106  -m "subset " print reports with this subset name
107  -k "gsi rtype " print reports with this gsi report type
108  -r "on29 rtype " print reports with this on29 report type
109  -d print reports using ufdump - note: this works with any NCEP BUFR file
110  -n no pause between reports output
111  -h print only report headers
112 
113  Only a filename is required in which case step through the reports one at a time using "enter"
114 
115  Optional arguments can also be applied in the pause between reports output without using a dash
116 
117  Optional arguments will be applied in concert in most cases
118 </pre>

311 ### cmpbqm
312 
313 An inventory of prepbufr observations by variable, report type, and quality mark made from a prepbufr file. The ob type,total count,and quality marks are listed by column. The cka and ckb columns are counts of observed values with missing qm, or qms with missing observations. The cka and ckb should be zero but sometimes they're not. The qm values are found in [bufr code tables](https://www.emc.ncep.noaa.gov/mmb/data_processing/prepbufr.doc/table_7.htm), but below a quick summary. The GSI qms are added by a program run by the `fit2obs` system which copies that information from the convstat files. The sample output prepbufr was after prep but pre-analysis.
314 |Quality Marker| Description|
315 |:--:|-----------|
316 |0 | keep flag|
317 |1 | passed checks and/or corrected by cqc|
318 |2 | not checked|
319 |3 | suspicious|
320 |4-7 | rejected by oiqc (original mark + 4)|
321 |8 | Ps more than 100mb off|
322 |9 | filtered by missing ob errors in gsi error table|
323 |10 | rejected by gsi gross check|
324 |11 | rejected by gsi varqc|
325 |12 | on manual reject list|
326 |13 | rejected by cqc or acqc|
327 |14 | sdm reject (manual purge)|
328 |15 | rejected by prepdata code (ie failed various sanity checks)|
329 |cka | a non-missing value with a missing quality mark|
330 |ckb | a missing value with a non-missing quality mark|
331 
332 See the source code at cmpbqm.f90

552 ### xbfmg
553 
554 This program splits a single file containing one or more BUFR messages into one or more
555 BUFR files each containing a single BUFR message. The output BUFR files are written to the
556 current working directory, according to a pre-defined naming convention as described below.
557 
558 <pre>
559 Usage:
560 
561  xbfmg [-v] [-h] [-g] bufrfile
562 
563  where:
564  -v prints version information and exits
565 
566  -h prints program help and usage information and exits
567 
568  -g preserves within each output file any GTS bulletin header and control
569  characters associated with the corresponding BUFR message from the
570  input file
571 
572  bufrfile [path/]name of input file containing one or more BUFR messages to be
573  extracted into separate output files within the current working directory
574 
575  The output will be stored within the current working directory using the
576  following filenames:
577 
578  (basename).xbfmg.out.000001
579  (basename).xbfmg.out.000002
580  (basename).xbfmg.out.000003
581  and so on, up through
582  (basename).xbfmg.out.(last#)
583 
584  where:
585 
586  (basename) = basename of bufrfile
587 
588  (last#) = total number of BUFR messages in bufrfile
589 </pre>

591 See the source code at xbfmg.c
